# ClawOS-Zhongtai

**把 Agent 组织成一个真正能运转的中台。**

ClawOS-Zhongtai 是一个面向多 Agent 组织协作的产品原型。

它想解决的，不是“再多做几个 agent”，而是更难、也更现实的问题：

- 谁在负责什么
- 谁现在真的在工作
- 工作是怎么交接的
- 哪些能力可以被上架、被调用、被雇用
- 操作者怎样像管理一个组织，而不是盯着一堆聊天窗口

所以它不是一个单纯的 agent 面板。

它更像一个**组织操作界面**：
- 有总览
- 有 roster
- 有可被组织调用的员工 / 能力市场
- 也可以继续往三省六部这样的角色化组织皮肤去长

## 为什么现在值得做

今天很多团队已经不缺 agent 了。

真正开始缺的是：
- 组织结构
- 协作秩序
- 可见分工
- 能力编制
- 运营视角下的控制界面

也就是说，问题慢慢从“怎么把 agent 跑起来”，变成了：

**怎么把 agent 变成一个能被组织起来、被调度、被治理的系统。**

ClawOS-Zhongtai 想卡的，就是这一步。

## 适合谁
- 想做多 Agent 组织化协作的人
- 想把 agent 变成岗位、角色、可上架能力的人
- 想做三省六部 / 中台制 / 角色制产品界面的人
- 想让操作者不再靠记忆管理系统，而是靠结构管理系统的人

## 产品方向

ClawOS-Zhongtai 的方向，不是做一个更花哨的 AI 页面。

它更接近这几件事的交叉点：
- 多 Agent 组织协作
- 角色分工
- 任务承接与调度
- 能力市场化
- 可换皮肤的组织界面

三省六部会是它很自然的一层表达方式，
但不应该只是古风包装。

更准确地说：

**我们在做的是一个可以承载组织结构的 Agent 中台。**
