# ClawOS-Zhongtai

**把 Agent 组织成一个真正能运转的中台。**

> 别再让你的 agents 开群聊了，给他们一个真正的组织结构。

ClawOS-Zhongtai 是一个面向多 Agent 组织协作的产品原型。

它想解决的，不是“再多做几个 agent”，而是更难、也更现实的问题：

- 谁在负责什么
- 谁现在真的在工作
- 工作是怎么交接的
- 哪些能力可以被上架、被调用、被雇用
- 操作者怎样像管理一个组织，而不是盯着一堆聊天窗口

所以它不是一个单纯的 agent 面板。

它更像一个**组织操作界面**：
- 有总览
- 有 roster
- 有可被组织调用的员工 / 能力市场
- 也可以继续往三省六部这样的角色化组织皮肤去长

更重要的是：

**ClawOS-Zhongtai 不只是一个单独界面。它更像一个 AI 组织内核。**  
三省六部是第一套已经跑出来的表达层，后面还可以继续长出西游、后宫、仙班，以及更多可运行、可传播、可记住的主题组织包。

## 它和普通 agent dashboard 的区别
普通 agent dashboard 往往回答的是：
- 哪个模型在跑
- 哪段对话在动
- 哪个任务报错了

ClawOS-Zhongtai 更想回答的是：
- 这个组织里有哪些角色
- 谁在承担什么职责
- 能力怎样被编入组织
- 操作者如何像管理一个系统，而不是追踪一堆离散会话

也就是说，它不是只看“活动”，而是开始看**组织结构**。

## 当前核心模块
- **Dashboard**：系统总控视角
- **Roster**：组织 roster / 岗位视角
- **Marketplace**：能力上架与雇用视角
- **Theme Layer**：把三省六部等组织语言放到展示层，而不是写死在内核里

## 产品亮点
- **组织优先，不是 prompt 优先**：重点不是某个 agent 有多聪明，而是它如何被编入一个可管理的组织结构。
- **中台化视角**：把总览、岗位 roster、能力市场放到同一个产品叙事里，而不是拆成零碎工具。
- **更适合操作者**：它不是给人看一堆底层 activity，而是帮助人判断系统状态、组织能力和调度空间。
- **可主题化表达**：三省六部可以是天然表达方式，但它属于展示层，不该污染底层产品抽象。
- **能力可上架**：不是只有“agent 在跑”，而是开始出现“能力单元可被雇用、接入、编组”的产品语义。
- **一套内核，多套世界**：今天是三省六部，后面也可以是西游、后宫、仙班等不同组织语言的主题层。

## 使用场景
- **多 Agent 组织总控台**：让操作者看到一个组织，而不是一堆散会话。
- **员工 / 角色中台**：把 agent 视为岗位、角色、编制，而不只是 bot。
- **能力市场**：把已产品化能力做成可浏览、可接入、可雇用的界面。
- **三省六部式组织产品**：用更有秩序感的组织语言包装多 Agent 协作。
- **AI 团队产品化展示**：适合面向内部汇报、对外 demo、产品方向验证。

## FAQ
### 它是一个 agent framework 吗？
不是。它更像组织层 / 操作层 / 展示层产品，而不是底层模型框架本身。

### 它和普通控制台有什么不一样？
普通控制台更偏监控；ClawOS-Zhongtai 更偏组织结构、角色编排、能力市场和操作者视角。

### 三省六部是不是只是皮肤？
是表达层的一部分，但不是随便套个古风壳。更准确地说，它是一种组织语言，应该服务于结构，而不是替代结构。

### 它未来会只有三省六部一种形态吗？
不会。三省六部是很自然的一种主题方向，但长期看应该支持不同组织皮肤和角色体系。
