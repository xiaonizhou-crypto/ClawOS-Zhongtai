# Apply Plan

This file turns the Hougong Theme Pack from a reference pack into an implementation plan.

## Goal
Apply the Hougong theme onto a `sansheng`-style project while preserving the neutral governance kernel.

## Approach
Use a two-layer replacement strategy:

### Layer 1 — agent identity layer
Replace:
- `IDENTITY.md`
- `SOUL.md`
- avatar assets

Map by responsibility, not by literal title.

### Layer 2 — presentation layer
Replace:
- labels
- headings
- module subtitles
- role cards
- pack descriptions
- operator copy

Do not replace kernel variables or execution logic.

---

## Minimal real-world apply flow

1. Copy avatar assets into target project assets
2. Replace the mapped `IDENTITY.md` files
3. Replace the mapped `SOUL.md` files
4. Add theme docs into the target project for operator reference
5. Update visible UI naming only where presentation strings exist
6. Validate that roles still make sense under the original task / governance flow

## Definition of done
A project is considered successfully themed when:
- avatar layer is fully replaced
- all mapped roles have updated `IDENTITY.md`
- all mapped roles have updated `SOUL.md`
- visible role naming is theme-consistent
- user voice feels immersive within the theme
- structural clarity remains intact
- no kernel logic was rewritten just to fit the theme
